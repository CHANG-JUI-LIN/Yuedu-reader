# IDPF EPUB 3 Compatibility Matrix

This matrix records the first automated baseline/current capture from the
official IDPF EPUB 3 sample corpus. It is a triage snapshot, not a claim of
complete EPUB 3 support.

Capture identity:

- Baseline production: `build-week-baseline` at `dd62d80`.
- Current branch and harness: `codex/openai-build-week` at `fa95f83`.
- Device: iPhone 17 Pro Max simulator, iOS 27.0.
- Baseline and current: 43 runs, 29 passed, 14 failed, 0 skipped; all 42
  sample-level results are identical.
- Static scan: 42 passed in both captures.
- Manual review: the eight representative samples remain `not-run`.

`B` columns are baseline and `C` columns are current. Stage values are
`pass`, `fail`, `not-run`, or `n/a`. Fixed-layout and mixed-layout targets
use their capability-specific render path, so paged and scroll checks are
`n/a`. A `not-run` final outcome on the representative set means automated
evidence exists but the required manual checkpoints are still pending. Raw
JSON and `.xcresult` bundles remain in the ignored local results directory and
are not committed.
